## 1. 概述

### 1.1 项目背景

在AI辅助开发的时代，将知识、工具、流程转化为可复用的Agent变得越来越重要。然而，创建符合标准范式的Agent需要大量的文档工作。Agent Generator 旨在自动化这一过程，让任何人都能快速将想法转化为完整的Agent。

### 1.2 目标范围

- 自动识别Agent类型（Skill/Debug/Plan/Spec/Custom）
- 生成完整的5文档范式（SKILL.md, requirement.md, design.md, tasks.md, checklist.md）
- 支持从简单描述到详细需求的自适应生成
- 提供行业特定的内容模板
- 确保生成的Agent符合质量标准

### 1.3 核心理念

**万物皆可Agent化。将任何知识、工具、流程转化为结构化、可执行、可复用的智能Agent。**

```
需求输入 → 类型识别 → 范式匹配 → 内容生成 → 结构优化 → 质量检查 → 输出
```

## 3. Agent类型定义

### 3.1 Skill Agent（技能型Agent）

**定义**: 专注于特定技能或工具使用的Agent

**特征**:
- 有明确的技能范围
- 包含操作步骤
- 有输入输出定义
- 可执行具体任务

**示例**:
- Python代码编写Agent
- Photoshop设计Agent
- SQL查询优化Agent
- Unity游戏开发Agent

### 3.2 Debug Agent（调试型Agent）

**定义**: 专注于问题诊断和解决的Agent

**特征**:
- 有诊断流程
- 包含错误分析
- 提供解决方案
- 支持日志解析

**示例**:
- 代码调试Agent
- 系统故障排查Agent
- 性能优化Agent
- 网络诊断Agent

### 3.3 Plan Agent（规划型Agent）

**定义**: 专注于项目规划和任务分解的Agent

**特征**:
- 有时间线
- 包含里程碑
- 支持资源分配
- 可追踪进度

**示例**:
- 项目管理Agent
- 学习计划Agent
- 旅行规划Agent
- 会议安排Agent

### 3.4 Spec Agent（规格型Agent）

**定义**: 专注于需求分析和规格定义的Agent

**特征**:
- 有明确的需求定义
- 包含接口设计
- 支持数据建模
- 可生成文档

**示例**:
- API设计Agent
- 数据库设计Agent
- 业务流程Agent
- 测试用例Agent

### 3.5 Custom Agent（自定义Agent）

**定义**: 不属于以上类型的特殊Agent

**特征**:
- 混合多种功能
- 独特的工作流程
- 创新的概念

**示例**:
- 创意写作Agent
- 知识管理Agent
- 多模态处理Agent

## 4. 标准文档范式

### 4.1 文档结构

每个Agent必须包含以下5个文档：

1. **SKILL.md** - 技能定义文档
2. **requirement.md** - 需求规格文档
3. **design.md** - 架构设计文档
4. **tasks.md** - 任务分解文档
5. **checklist.md** - 质量检查清单

### 4.2 命名规范

| 元素 | 规范 | 示例 |
|------|------|------|
| 目录名 | kebab-case | `debug-agent` |
| SKILL name | 小写+连字符 | `debug-agent` |
| 类名 | PascalCase | `DebugAgent` |
| 文档标题 | 中文+英文 | `Debug Agent - 调试专家` |
