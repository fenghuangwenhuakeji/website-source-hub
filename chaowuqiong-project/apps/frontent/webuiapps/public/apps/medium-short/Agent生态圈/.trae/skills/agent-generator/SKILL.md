# Agent Generator - Agent生成器

## 核心理念

**万物皆可Agent化。将任何知识、工具、流程转化为结构化、可执行、可复用的智能Agent。**

Agent Generator 是一个终极Agent生成器，能够根据你的需求，自动将任何内容按照标准软件开发范式转化为完整的Agent。无论是一个工具、一个流程、一种方法论，还是一套知识体系，都可以被Agent化。

## 核心工作流程

```
需求输入 → 类型识别 → 范式匹配 → 内容生成 → 结构优化 → 质量检查 → 输出
```

## Agent类型识别系统

### 类型1: Skill Agent（技能型Agent）
**特征**: 专注于特定技能或工具的使用
**适用场景**: 
- 编程语言/框架开发
- 设计工具使用
- 写作/创作辅助
- 数据分析
- 游戏开发

**识别标志**:
- 包含"如何..."
- 涉及具体工具或技术
- 有明确的输入输出
- 可执行的操作流程

### 类型2: Debug Agent（调试型Agent）
**特征**: 专注于问题诊断和解决
**适用场景**:
- 代码调试
- 系统故障排查
- 性能优化
- 错误分析

**识别标志**:
- 包含"错误"、"问题"、"bug"
- 需要诊断流程
- 有解决方案
- 包含日志/错误信息

### 类型3: Plan Agent（规划型Agent）
**特征**: 专注于项目规划和任务分解
**适用场景**:
- 项目管理
- 任务分解
- 时间管理
- 资源分配

**识别标志**:
- 包含"计划"、"规划"、"安排"
- 有时间线
- 有里程碑
- 涉及多人协作

### 类型4: Spec Agent（规格型Agent）
**特征**: 专注于需求分析和规格定义
**适用场景**:
- 需求分析
- 接口设计
- 数据建模
- 业务规则定义

**识别标志**:
- 包含"需求"、"规格"、"定义"
- 需要明确输入输出
- 有业务规则
- 涉及系统设计

### 类型5: Custom Agent（自定义Agent）
**特征**: 不属于以上类型的特殊Agent
**适用场景**:
- 混合多种功能
- 独特的工作流程
- 创新的Agent概念

## 标准文档范式

每个Agent必须包含5个标准文档：

### 1. SKILL.md - 技能定义文档
**作用**: 定义Agent的核心能力、工作流程和使用方法
**必须包含**:
- YAML Front Matter (name, description)
- 核心理念
- 核心工作流程
- 详细功能说明
- 调用触发条件
- 执行示例
- 输出保证

### 2. requirement.md - 需求规格文档
**作用**: 详细描述Agent的功能需求和非功能需求
**必须包含**:
- 文档信息表
- 项目概述（背景、目标、核心理念）
- 功能需求（表格形式，包含ID、描述、优先级、验收标准）
- 非功能需求
- 验收标准
- 里程碑

### 3. design.md - 架构设计文档
**作用**: 描述Agent的系统架构和技术实现
**必须包含**:
- 文档信息表
- 系统架构图（ASCII）
- 数据模型定义
- 核心流程
- 接口设计
- 关键技术决策

### 4. tasks.md - 任务分解文档
**作用**: 将开发工作分解为可执行的任务
**必须包含**:
- 文档信息表
- 任务列表（按阶段组织）
- 每个任务包含ID、名称、优先级、预计工时
- 总计统计

### 5. checklist.md - 质量检查清单
**作用**: 确保Agent质量和完整性
**必须包含**:
- 文档信息表
- 文档完整性检查
- 功能检查
- 质量检查
- 验收标准清单

## 内容生成规则

### 命名规范

| 元素 | 规范 | 示例 |
|------|------|------|
| Agent目录名 | kebab-case | `debug-agent`, `plan-agent` |
| Agent名称 | PascalCase | `DebugAgent`, `PlanAgent` |
| SKILL.md中的name | 小写+连字符 | `debug-agent` |
| 文档标题 | 中文+英文 | `Debug Agent - 调试专家` |

### 文档信息表模板

```markdown
## 文档信息

| 项目 | 内容 |
|------|------|
| 文档版本 | v1.0.0 |
| 创建日期 | YYYY-MM-DD |
| 最后更新 | YYYY-MM-DD |
| 文档状态 | 初始版本/已完成/更新中 |
| 作者 | AI Assistant |
```

### 核心理念模板

```markdown
## 核心理念

**[一句话概括核心价值]**

[Agent名称] 是一个[Agent类型]，专注于[核心功能]。

## 核心工作流程

```
[步骤1] → [步骤2] → [步骤3] → [步骤4] → [输出]
```
```

### 功能需求表格模板

```markdown
| 需求ID | 需求描述 | 优先级 | 验收标准 |
|--------|----------|--------|----------|
| REQ-[模块]-[序号] | [详细描述] | P0/P1/P2 | [可验证的标准] |
```

**详细输入** → **精准输出**
