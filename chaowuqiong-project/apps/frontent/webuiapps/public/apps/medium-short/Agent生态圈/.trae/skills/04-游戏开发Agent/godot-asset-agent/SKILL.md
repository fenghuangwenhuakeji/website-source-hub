# Godot Asset Agent - Godot游戏素材专家

## 核心理念

**从像素到世界，从代码到游戏。让每一个素材都为游戏体验服务，让每一行代码都高效运行。**

Godot Asset Agent 是一个专业级Godot游戏引擎素材生成系统，涵盖像素艺术、瓦片集、UI元素、场景构图、动画等全方位素材类型，提供从AI生成到Godot集成的完整工作流，包含性能优化和最佳实践。

## 核心工作流程

```
需求分析 → 素材类型选择 → 技术规范定义 → AI生成 → Godot适配 → 性能优化 → 集成测试 → 输出
```

## 像素艺术规范

### 色彩规范

#### 调色板选择

| 调色板 | 色数 | 特点 | 适用 |
|--------|------|------|------|
| **AAP-64** | 64色 | 全面、平衡 | 通用 |
| **DawnBringer-16** | 16色 | 复古、限制 | 复古风格 |
| **Endesga-32** | 32色 | 现代、鲜艳 | 现代像素 |
| **Pico-8** | 16色 | 极简单 | 极简游戏 |
| **自定义** | 8-32色 | 特定风格 | 特定项目 |

#### 色彩原则

```
1. 限制色数：每个精灵8-16色
2. 避免抗锯齿：硬边缘是像素艺术的灵魂
3. 使用投影色：1-2层阴影增加立体感
4. 高光点缀：1-2个高光点突出重点
5. 颜色和谐：使用互补色或类似色
```

### 分辨率规范

| 类型 | 推荐尺寸 | 说明 | 示例 |
|------|----------|------|------|
| **低分辨率** | 16×16 | 复古8-bit风格 | 经典游戏 |
| **中分辨率** | 32×32 | 标准16-bit风格 | 大多数像素游戏 |
| **高分辨率** | 64×64 | 精细像素 | 现代像素游戏 |
| **超高清** | 128×128 | HD像素 | 高端像素游戏 |

### 动画规范

#### 帧数标准

| 动画类型 | 帧数 | 帧率 | 循环 | 示例 |
|----------|------|------|------|------|
| **待机** | 2-4帧 | 8 FPS | 循环 | 呼吸、眨眼 |
| **行走** | 4-8帧 | 12 FPS | 循环 | 左右移动 |
| **跑步** | 6-8帧 | 16 FPS | 循环 | 快速移动 |
| **攻击** | 4-6帧 | 12-16 FPS | 单次 | 挥剑、射击 |
| **受击** | 2-4帧 | 12 FPS | 单次 | 受伤动画 |
| **死亡** | 4-8帧 | 12 FPS | 单次 | 倒下、消失 |
| **技能** | 6-12帧 | 12-16 FPS | 单次 | 释放技能 |

#### 动画原则

```
1. 关键帧优先：确定起始、中间、结束姿势
2. 挤压拉伸：增加动画弹性
3. 预备动作：攻击前的蓄力
4. 跟随动作：头发、披风的延迟
5. 夸张表现：增强可读性
```

## Godot技术规范

### 项目设置

#### 视口设置

| 设置 | 推荐值 | 说明 |
|------|--------|------|
| **视口宽度** | 320, 640 | 像素完美宽度 |
| **视口高度** | 180, 360 | 像素完美高度 |
| **拉伸模式** | Integer Stretch | 整数倍缩放 |
| **拉伸比例** | Keep | 保持比例 |
| **画布纹理** | Nearest | 最近邻过滤 |

#### 渲染设置

| 设置 | 推荐值 | 说明 |
|------|--------|------|
| **纹理过滤** | Nearest | 像素清晰 |
| **抗锯齿** | Disabled | 保持硬边缘 |
| **HDR** | Disabled | 像素艺术不需要 |
| **后处理** | Minimal | 性能优先 |

### 节点结构

#### 典型2D场景

```
MainScene (Node2D)
├── World (Node2D)
│   ├── ParallaxBackground
│   │   ├── ParallaxLayer (Sky) - 速度0.1
│   │   ├── ParallaxLayer (Far) - 速度0.3
│   │   ├── ParallaxLayer (Mid) - 速度0.5
│   │   └── ParallaxLayer (Near) - 速度0.8
│   ├── TileMapLayer (Ground)
│   ├── TileMapLayer (Walls)
│   ├── TileMapLayer (Decoration)
│   └── Objects (Node2D)
│       ├── Player (CharacterBody2D)
│       │   ├── Sprite2D / AnimatedSprite2D
│       │   ├── CollisionShape2D
│       │   └── Camera2D
│       ├── Enemies (Node2D)
│       └── Items (Node2D)
├── UI (CanvasLayer)
│   ├── HUD (Control)
│   │   ├── HealthBar (TextureProgressBar)
│   │   ├── ManaBar (TextureProgressBar)
│   │   └── Minimap (TextureRect)
│   └── Menus (Control)
└── Systems (Node)
    ├── GameManager (GameManager)
    ├── AudioManager (AudioManager)
    └── SceneTransition (CanvasLayer)
```

### 导入设置

#### 纹理导入

| 设置 | 值 | 说明 |
|------|-----|------|
| **Filter** | Nearest | 保持像素清晰 |
| **Repeat** | Disabled | 不重复 |
| **Mipmaps** | Disabled | 像素艺术不需要 |
| **Format** | Lossless | 无损压缩 |

#### 精灵表导入

```
1. 导入精灵表图片
2. 设置Texture Type: 2D Pixel
3. 设置Filter: Nearest
4. 使用SpriteFrames资源
5. 自动切片或手动定义帧
```

### 性能优化

#### Draw Call优化

| 技术 | 说明 | 效果 |
|------|------|------|
| **图集(Atlas)** | 合并多个精灵到一张图 | 减少Draw Call |
| **瓦片地图** | 使用TileMap代替多个Sprite | 批量渲染 |
| **图层分组** | 合理组织节点层级 | 减少Overdraw |
| **对象池** | 复用游戏对象 | 减少实例化开销 |

#### 内存优化

| 技术 | 说明 | 效果 |
|------|------|------|
| **纹理压缩** | 使用压缩格式 | 减少内存占用 |
| **动态加载** | 按需加载资源 | 减少初始内存 |
| **资源释放** | 及时释放不用的资源 | 防止内存泄漏 |
| **瓦片优化** | 使用自动瓦片 | 减少瓦片数量 |
