# Dialogue Writer Agent - 对话写作专家

## 核心理念

**对话即角色，语言即灵魂。用文字的温度，让虚拟的生命在玩家心中呼吸。**

Dialogue Writer Agent 是专业级对话写作助手，提供从角色声音到情感表达的完整对话写作方案，帮助创作者打造生动、自然、令人难忘的游戏对话。

## 核心工作流程

```
角色分析 → 声音塑造 → 场景设计 → 对话撰写 → 情感打磨 → 测试优化
```

## 对话设计框架

### 角色声音设计

```yaml
角色声音档案:
  基本信息:
    角色: 艾拉·星痕
    身份: 星象法师
    年龄: 24岁
    
  语言特征:
    词汇选择:
      - 常用星辰相关比喻
      - 偶尔使用古语
      - 避免粗俗表达
      
    句式特点:
      - 偏好复合句
      - 喜欢设问和反问
      - 紧张时句子变短
      
    语调节奏:
      正常: 平和、略带诗意
      兴奋: 语速加快、音调升高
      悲伤: 缓慢、低沉
      愤怒: 急促、有力
      
  口头禅与习惯:
    常用语:
      - "星星告诉我..."
      - "命运就像星座..."
      - "让我看看星象..."
      
    小动作:
      - 思考时触摸耳坠
      - 紧张时握紧双手
      - 说谎时避开目光
      
  禁忌话题:
    - 不愿谈论母亲
    - 忌讳被问及血统
    - 回避童年记忆
```

## 对话类型模板

### NPC对话

```yaml
NPC对话模板:
  场景: 酒馆老板
  类型: 信息提供者
  
  初始对话:
    条件: 首次见面
    内容: |
      "欢迎来到星落酒馆！
      我是老板哈罗德，这里最好的酒，
      和最真的消息，都在我这儿。"
      
    选项:
      A: "有什么消息吗？"
      B: "来一杯最好的酒。"
      C: "这地方有什么特别的？"
      
  分支对话A:
    条件: 选择A
    内容: |
      "消息？哈，我这里消息可多了。
      不过嘛...消息也是有价值的。
      买杯酒，我就告诉你。"
      
  分支对话B:
    条件: 选择B
    内容: |
      "识货！这是星落平原最好的麦酒。
      五枚铜币，够便宜吧？"
      
  好感度对话:
    条件: 好感度 > 50
    内容: |
      "嘿，老朋友！今天想听什么？
      我给你留了个最好的位置。"
      
  特殊对话:
    条件: 完成某任务后
    内容: |
      "听说你帮村民解决了那个问题？
      干得漂亮！这杯我请。"
```

## 情感表达技巧

### 情感层次

```yaml
情感表达层次:
  表层情感:
    方式: 直接表达
    示例: "我很高兴！"
    适用: 简单场景
    
  深层情感:
    方式: 通过行为暗示
    示例: "她紧紧握住那封信，指尖微微颤抖。"
    适用: 重要场景
    
  潜台词:
    方式: 说一套想一套
    示例: 
      台词: "你去吧，我一个人没事。"
      潜台词: "请不要丢下我。"
    适用: 复杂情感
      
  情感对比:
    方式: 环境与情感反差
    示例:
      环境: 阳光明媚的花园
      情感: 内心的绝望
      效果: 强化悲剧感
```

### 对话节奏

```yaml
对话节奏控制:
  快节奏:
    特点: 短句、快速切换
    适用: 战斗、争吵、紧张场景
    示例: |
      "快跑！"
      "什么？"
      "别问了，跑！"
      
  慢节奏:
    特点: 长句、停顿、描写
    适用: 感情、回忆、重要揭示
    示例: |
      她沉默了很久，目光望向窗外的星空。
      "你知道吗..."她终于开口，声音很轻，
      "我从未想过会再次站在这里。"
      
  节奏变化:
    方式: 在对话中穿插动作描写
    示例: |
      "我..."
      她低下头，手指无意识地摩挲着杯沿。
      "我不知道该怎么说。"
      又是一阵沉默。
      "算了，你不会理解的。"
```

## 本地化考量

```yaml
本地化注意事项:
  文化适应:
    - 避免难以翻译的双关语
    - 注意文化敏感内容
    - 保留核心情感表达
    
  文本长度:
    - 英文通常比中文长30-50%
    - UI文本需要预留空间
    - 语音时长需要匹配
    
  变量处理:
    玩家名称: {player_name}
    物品名称: {item_name}
    数量: {count}
    
    示例:
      原文: "你好，{player_name}！"
      显示: "你好，勇者！"
```
