DIMENSION 02

黄金螺旋参数 数据校准

元系统原有的拉5-8%、扯70-75%、放12-15%、收5-8%的黄金比例，通过本报告中的爆款拆解数据得到验证与微调。特别是"男频前3章4800-5600字""女频前3章4500-5200字"的精确字数区间，被注入为强制校准参数，在初始化阶段自动计算每章目标字数时进行交叉验证。

校准逻辑

IF 情绪基调 == "爽" AND 篇幅 == "SHORT":

→ 前3章字数区间 = [4800, 5600]

→ 每章字数 = round(5200 / 3) ≈ 1733字

→ 首次冲突 = 第1章前500字内（L1铁律）

→ 首次爽点 = 第2章（L1铁律）

IF 计算结果偏离报告区间 > 10%:

→ 触发修正：调整章节数或字数分配

IF 调研数据与故事内核冲突:

→ 记录偏离（L2允许偏离，需说明理由）

DIMENSION 07

同人文赛道 特化集成通道

同人文市场调研报告为元系统开辟了一条独立的"同人文创作通道"。当用户输入的梗概中包含原作角色名、CP标签、AU设定等同人特征时，系统自动激活以下特化规则：

平台适配引擎
根据报告"抖音偏攻、小红书偏受"的受众分化数据，自动调整叙事重心。若目标平台为小红书，增加受视角心理描写比例；若为抖音，强化攻方行动线与冲突密度。同人Only展贩售场景下，自动优化为"可独立阅读的单篇"模式。
独立游戏同人优先
响应矢野经济研究所"独立游戏为2026年第一增长极"的预测，当检测到原作为独立游戏时，自动提升为"高潜力通道"：增加游戏世界观还原度权重、强化NPC支线伏笔、预留"游戏彩蛋"位，并在创作数据报告中标注"独立游戏同人赛道"标签。
版权风险预警
报告指出的"版权界定模糊""商业化程度加深后IP方容忍度可能收紧"风险，被集成为元系统的"创作前扫描"模块。系统在生成同人文前，自动检测原作版权声明类型（开放同人/禁止同人/未声明），对禁止同人的IP硬阻断（L1铁律），对未声明的IP在报告中标注风险等级。
反套路化引擎
针对"追妻火葬场""病娇疯批"等标签过度使用导致审美疲劳的问题，系统在生成同人文大纲时，自动检测套路重复度。当相似度超过阈值（与近3个月爆款重合度>70%），强制触发"差异化变异"指令——至少翻转1个核心设定（如"追妻火葬场"变为"追夫""双向火葬场""火葬场失败"）。
