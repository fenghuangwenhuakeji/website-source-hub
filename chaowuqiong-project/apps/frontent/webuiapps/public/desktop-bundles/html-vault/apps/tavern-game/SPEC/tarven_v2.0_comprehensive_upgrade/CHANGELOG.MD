## 📝 版本规范

### 版本号格式
```
v{大版本}.{迭代轮次}.{修订}
例如: v2.0.1
```

- **大版本**: 重大架构调整、V2.0全面升级 (v2.x)
- **迭代轮次**: 每轮迭代递增 (v2.0.x → v2.1.x)
- **修订**: 小修改、文档更新 (v2.0.0 → v2.0.1)

### 文档版本策略
- **REQUIREMENT**: 每轮需求迭代后迭代轮次+1
- **DESIGN**: 设计调整后迭代轮次+1
- **TASK**: 任务更新后迭代轮次+1
- **同步原则**: 三个文档的迭代版本应保持一致
