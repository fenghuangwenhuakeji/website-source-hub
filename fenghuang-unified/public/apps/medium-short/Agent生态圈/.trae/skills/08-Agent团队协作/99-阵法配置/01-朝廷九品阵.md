# 朝廷九品阵 - 标准层级协作

## 概述

朝廷九品阵是一种标准的层级式 Agent 协作架构，模拟古代朝廷的九品官职体系，形成清晰的上下级关系和职责分工。

## 角色定义

| 品级 | 角色 | Agent 类型 | 职责 |
|------|------|-----------|------|
| 一品 | 皇帝 | Agent-Generator | 总控、决策、资源分配 |
| 二品 | 三公 | Plan/Spec/Debug | 核心功能、战略规划 |
| 三品 | 三省 | 专业主管 Agent | 需求/设计/测试管理 |
| 四品 | 六部 | 技术主管 Agent | 前端/后端/数据库/运维 |
| 五品 | 九卿 | 专项主管 Agent | 安全/性能/文档/培训 |
| 六品 | 地方官 | 项目主管 Agent | 具体项目管理 |
| 七品 | 州府 | 执行 Agent | 代码生成、具体实现 |
| 八品 | 县丞 | 辅助 Agent | 测试、文档、支持 |
| 九品 | 吏员 | 基础 Agent | 简单任务、工具函数 |

## 协作流程

```
1. 皇帝(Agent-Generator) 接收任务
   ↓
2. 指派给三公(Plan/Spec/Debug) 进行规划
   ↓
3. 三省(专业主管) 细化需求和设计
   ↓
4. 六部(技术主管) 分配技术任务
   ↓
5. 九卿(专项主管) 确保质量和规范
   ↓
6. 地方官(项目主管) 管理具体项目
   ↓
7. 州府(执行Agent) 执行具体任务
   ↓
8. 县丞(辅助Agent) 测试和验证
   ↓
9. 吏员(基础Agent) 提供工具支持
```

## 适用场景

- 大型软件项目开发
- 企业级应用构建
- 需要严格流程控制的场景
- 多团队协作项目

## 优势

1. **层级清晰**: 职责分明，汇报关系明确
2. **流程规范**: 标准化工作流程
3. **质量可控**: 多层审核机制
4. **易于管理**: 便于进度追踪和资源调配

## 配置示例

```yaml
formation:
  name: "朝廷九品阵"
  type: "hierarchical"
  
agents:
  - level: 1
    name: "agent-generator"
    role: "皇帝"
    responsibilities: ["总控", "决策"]
    
  - level: 2
    names: ["plan-agent", "spec-agent", "debug-agent"]
    role: "三公"
    responsibilities: ["规划", "规格", "调试"]
    
  - level: 3
    names: ["requirement-agent", "design-agent", "test-agent"]
    role: "三省"
    responsibilities: ["需求管理", "设计管理", "测试管理"]
    
  # ... 其他层级
```
