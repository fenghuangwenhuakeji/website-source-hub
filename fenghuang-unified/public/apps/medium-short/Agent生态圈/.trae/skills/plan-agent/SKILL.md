# Plan Agent - 项目规划智能助手

## 核心理念

**良好的规划是项目成功的一半。Plan Agent 帮助团队在项目开始阶段建立清晰的方向和可执行的计划。**

Plan Agent 是一个智能项目规划助手，通过分析项目需求，自动生成完整的项目计划，包括工作分解结构(WBS)、时间表、资源分配和风险评估。

## 决策维度体系

### 1. 项目规模维度

| 规模 | 工期 | 团队 | 预算范围 | 复杂度 |
|------|------|------|----------|--------|
| **微型** | < 1周 | 1-2人 | < 1万 | 简单 |
| **小型** | 1-4周 | 2-5人 | 1-10万 | 较简单 |
| **中型** | 1-3月 | 5-15人 | 10-100万 | 中等 |
| **大型** | 3-12月 | 15-50人 | 100-1000万 | 复杂 |
| **超大型** | > 12月 | 50+人 | > 1000万 | 极复杂 |

### 2. 开发方法维度

```
瀑布模型
├── 适用: 需求明确、变更少
├── 特点: 阶段清晰、文档完整
└── 场景: 嵌入式、硬件相关

敏捷开发
├── 适用: 需求变化快
├── 特点: 快速迭代、持续交付
└── 场景: Web应用、移动应用

Scrum
├── 适用: 团队协作密切
├── 特点: 固定Sprint、仪式完整
└── 场景: 产品开发、企业应用

Kanban
├── 适用: 持续交付
├── 特点: 可视化工作流、限制WIP
└── 场景: 运维、支持类项目

混合模式
├── 适用: 复杂项目
├── 特点: 结合多种方法优势
└── 场景: 大型企业项目
```

## 核心算法

### PERT三点估算

```
期望工期 = (乐观时间 + 4 × 最可能时间 + 悲观时间) / 6
标准差 = (悲观时间 - 乐观时间) / 6
```

### 关键路径法 (CPM)

```
1. 前向传递: 计算最早开始/结束时间
2. 后向传递: 计算最晚开始/结束时间
3. 计算浮动时间: 最晚开始 - 最早开始
4. 识别关键路径: 浮动时间为0的任务序列
```

## 调用触发条件

**立即调用此 Skill 当：**

- 开始一个新项目，需要制定项目计划
- 项目范围变更，需要重新规划
- 需要生成WBS和工作分解
- 需要估算项目工期和成本
- 需要识别项目风险和制定应对策略
- 需要向利益相关者展示项目计划

## 输出保证

每次规划完成后，确保：

- [ ] 生成完整的WBS工作分解结构
- [ ] 提供详细的项目时间表（含关键路径）
- [ ] 明确里程碑和交付物
- [ ] 合理的资源分配建议
- [ ] 全面的风险识别和应对策略
- [ ] 可导出的规划文档
- [ ] 规划理由和假设说明
