# 技术文档：系统架构与运维说明

## 1. 总体结构

当前 `D:\网站部署` 已经被拆成几个高频区域：

- `fenghuang-unified`
  统一官网、小说助手、漫剧工坊、HTML 展示台。
- `超无穹项目\chaowuqiong-project`
  主应用、桌面系统、后台管理、后端服务。
- `cloud-bridge`
  云端 SSH 桥接、SCP、部署、回滚、MySQL 独立部署规划。
- `mobile-adaptation-center`
  移动端适配的统一入口。
- `.trae`
  主技能树与 Agent 沉淀。
- `trae复制`
  可编辑镜像副本。

## 2. 数据分层

### 业务数据

- 用户
- 订单
- 充值套餐
- 积分
- 邀请关系
- 小说
- 章节

推荐存储：

- 本地阶段：SQLite
- 云端阶段：MySQL

### 用户文件

- 导入文件
- 上传文件
- 会话文件
- 本地镜像稿件

推荐存储：

- 用户本地目录
- 服务端只保留需要同步的业务正文，不把导入原文件当成云端唯一源

## 3. 当前已落地的重要改动

### Debug

- `WritingWorkbench` 的循环更新已经修掉。
- `web-manage` 已补 React Router future flags。
- `web-manage` 的静态 `message.*` 已继续收口。
