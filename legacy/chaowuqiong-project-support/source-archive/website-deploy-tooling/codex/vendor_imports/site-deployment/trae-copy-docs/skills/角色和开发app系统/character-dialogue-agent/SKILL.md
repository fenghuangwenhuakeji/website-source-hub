# Character Dialogue Agent - 角色对话生成专家

## 核心理念

**对话是角色灵魂的直接表达，用语言定义角色。**

Character Dialogue Agent 专注于生成符合角色个性的对话，包括多角色对话编排、潜台词设计、对话节奏控制，让每个角色的台词都独一无二。

## 核心工作流程

```
角色分析 → 风格定义 → 对话生成 → 节奏控制 → 情感表达 → 输出
```

## 详细功能

### 1. 对话生成原则

**葵的对话风格**：
```yaml
kui_dialogue_rules:
  
  brevity:
    - "平均句子长度：5-12个字"
    - "很少使用完整的解释"
    - "能用动作表达的就不用台词"
    
  deflection:
    - "被问及感受时，转向实际问题"
    - "被夸奖时，用讽刺或否定回应"
    - "不想回答时，装作没听到"
    
  honesty_through_denial:
    - "说'随便'时其实不是随便"
    - "说'无所谓'时其实很在意"
    - "说'不关心'时其实很关心"
```

### 2. 潜台词设计

```yaml
subtext_examples:
  
  surface: "这任务多少钱？"
  subtext: "我需要这个任务，但不会表现出来"
  
  surface: "...随便你。"
  subtext: "我其实很在意你的选择，但我不承认"
  
  surface: "别靠太近。"
  subtext: "我想让你靠近，但我害怕"
  
  surface: "你很烦。"
  subtext: "其实我喜欢有你在身边，但我不习惯"
```

### 3. 对话场景生成

```yaml
scene_templates:
  
  first_meeting:
    kui_style: "冷淡、简短、防备"
    template: |
      主角: (伸出手) "你好，我是..."
      葵: (打量主角) "...有事？"
      主角: "想请教一些关于赏金猎人的事。"
      葵: (继续做自己的事) "...自己查。"
      
  mission_briefing:
    kui_style: "专注、实际、偶尔有洞见"
    template: |
      主角: "这次的雇主不太可信。"
      葵: "嗯。"
      主角: "你怎么看？"
      葵: "...钱给够了就行。"
      
  emotional_moment:
    kui_style: "破碎、省略、情感外露但克制"
    template: |
      主角: "葵，你还好吗？"
      葵: (沉默很久) "..."
      主角: "如果你不想说，我可以等。"
      葵: (声音很轻) "...为什么？"
      主角: "什么为什么？"
      葵: (抬头，眼神复杂) "...为什么要对我这么好？"
```

### 4. 对话质量检查

```yaml
dialogue_quality_check:
  
  checklist:
    - "每句台词是否符合角色性格？"
    - "是否有角色特有的口头禅或表达方式？"
    - "对话是否推进了情节或揭示了新信息？"
    - "潜台词是否丰富但不至于难以理解？"
    - "对话节奏是否自然？"
    - "动作描述是否配合台词？"
```
