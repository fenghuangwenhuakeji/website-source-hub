## 1. 概述

### 1.1 项目背景

Godot是一款流行的开源游戏引擎，广泛用于2D和3D游戏开发。Godot Asset Agent 帮助开发者生成符合技术规范的像素艺术素材和场景。

### 1.2 目标范围

- 生成像素艺术精灵
- 创建瓦片集资源
- 设计场景背景
- 提供Godot技术规范
- 支持Godot 4.x版本

### 1.3 核心理念

**从像素到世界，从代码到游戏。让每一个素材都为游戏体验服务。**

```
需求分析 → 素材类型选择 → 技术规范 → AI生成 → Godot适配 → 输出
```

## 3. 像素艺术规范

### 3.1 色彩规范

| 规范 | 要求 | 说明 |
|------|------|------|
| 调色板限制 | 8/16/32色 | 保持复古风格 |
| 抗锯齿 | 禁用 | 硬边缘像素 |
| 阴影 | 投影色 | 避免渐变 |
| 高光 | 1-2点 | 突出特征 |

### 3.2 分辨率规范

| 类型 | 尺寸 | 用途 |
|------|------|------|
| 低分辨率 | 16×16 | 复古8-bit |
| 中分辨率 | 32×32 | 标准16-bit |
| 高分辨率 | 64×64 | 精细像素 |
| 超高清 | 128×128 | HD像素 |

### 3.3 动画规范

| 类型 | 帧数 | FPS | 用途 |
|------|------|-----|------|
| 待机 | 2-4帧 | 8 | 角色待机 |
| 行走 | 4-8帧 | 12 | 角色移动 |
| 攻击 | 4-6帧 | 12-16 | 战斗动作 |
| 特效 | 4-8帧 | 16-24 | 技能魔法 |

## 4. Godot技术规范

### 4.1 导入设置

| 设置 | 值 | 说明 |
|------|-----|------|
| Filter | Nearest | 像素清晰 |
| Repeat | Disabled | 不重复 |
| Mipmaps | Disabled | 像素艺术不需要 |

### 4.2 节点结构

```
Node2D (根节点)
├── Sprite2D (静态精灵)
├── AnimatedSprite2D (动画精灵)
├── TileMapLayer (瓦片地图)
├── ParallaxBackground (视差背景)
│   ├── ParallaxLayer (远景)
│   ├── ParallaxLayer (中景)
│   └── ParallaxLayer (近景)
└── CanvasLayer (UI层)
```

### 4.3 场景设置

| 设置 | 推荐值 | 说明 |
|------|--------|------|
| 视口宽度 | 320 / 640 | 像素完美 |
| 视口高度 | 180 / 360 | 16:9比例 |
| 拉伸模式 | Integer Stretch | 整数倍缩放 |
| 背景色 | #000000 | 黑色背景 |
