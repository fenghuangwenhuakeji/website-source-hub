## 1. 概述

### 1.1 项目背景

在软件开发过程中，选择合适的编程语言是项目成功的关键因素之一。不同的项目类型、规模、目标平台和团队背景都需要不同的技术栈。Language Selector Agent 是一个智能编程语言选择助手，旨在帮助开发者根据项目需求快速、准确地选择最适合的编程语言和技术栈。

### 1.2 目标范围

- 支持 12+ 种主流编程语言的选择分析
- 覆盖 Web、桌面、移动、嵌入式等多平台场景
- 考虑项目规模、团队背景、性能需求等多维度因素
- 提供技术栈组合推荐
- 生成详细的选择报告和理由

### 1.3 核心理念

**任何项目开始之前，都应该先明确需求，然后让智能体推荐最合适的技术栈。**

```
需求收集 → 维度分析 → 候选筛选 → 综合评分 → 最终推荐
```

```
桌面应用
├── Windows
├── macOS
├── Linux
└── 跨平台

系统/工具
├── 操作系统
├── 开发工具
├── 脚本工具
└── 嵌入式
```

### 4.2 项目规模维度

| 规模 | 代码行数 | 团队规模 | 典型项目 |
|------|----------|----------|----------|
| 微型 | < 1K | 1人 | 脚本、工具、原型 |
| 小型 | 1K-10K | 1-3人 | 个人项目、小工具 |
| 中型 | 10K-100K | 3-10人 | 团队项目、商业应用 |
| 大型 | 100K-1M | 10-50人 | 企业级系统 |
| 超大型 | > 1M | 50+人 | 操作系统、大型平台 |

### 4.3 目标用户维度

- **个人开发者/学习者**: 学习编程、个人项目、技术探索
- **学术研究**: 算法研究、数据分析、系统研究、教学演示
- **商业应用**: 初创公司、中小企业、大型企业、金融科技
- **政府/公共部门**: 政务系统、数据平台、基础设施

### 4.4 技术架构维度

- **前后端分离**: 前端 + 后端 + 数据库
- **全栈框架**: Next.js、Nuxt.js等
- **桌面应用**: 本地/远程API + 数据库
- **微服务**: 分布式架构
- **无服务器**: Serverless架构
- **单机应用**: 绿色软件

### 4.5 网络连接维度

- **完全离线**: 单机游戏、桌面工具、嵌入式系统
- **偶尔联网**: 同步数据、更新检查、云存档
- **始终在线**: Web应用、在线游戏、实时协作
- **混合模式**: PWA、离线优先、边缘计算
