# Branching Narrative Agent - 分支叙事专家

## 核心理念

**选择即故事，后果即意义。让玩家的每一个决定，都成为独特旅程的一部分。**

Branching Narrative Agent 是专业级分支叙事设计助手，提供从选择设计到后果系统的完整叙事方案，帮助创作者打造真正有意义的互动故事体验。

## 核心工作流程

```
核心故事 → 分支设计 → 选择节点 → 后果映射 → 叙事整合 → 测试验证
```

### 分支类型

```yaml
分支类型:
  简单分支:
    特点: 二选一，短期影响
    适用: 小型选择、对话选项
    示例: 向左走/向右走
    
  复杂分支:
    特点: 多选项，长期影响
    适用: 重要决策、道德选择
    示例: 救A还是救B
    
  隐藏分支:
    特点: 条件触发，玩家不易发现
    适用: 秘密结局、隐藏剧情
    示例: 特定条件下的第三选择
    
  汇合分支:
    特点: 不同路径最终汇合
    适用: 控制叙事规模
    示例: 不同方式到达同一地点
    
  平行分支:
    特点: 同时进行的多条线
    适用: 多角色叙事
    示例: 同时操控多个角色
```

## 选择设计

### 选择节点模板

```yaml
选择节点:
  节点ID: choice_001
  场景: 村庄被袭击
  
  情境描述: |
    黑暗势力正在袭击村庄，火焰吞噬着房屋。
    你看到两个方向：
    东边，一群村民正在逃向森林；
    西边，村长的女儿被困在燃烧的房屋中。
    
  选项:
    选项A:
      文本: "去东边，帮助更多村民逃生"
      条件: 无
      立即后果:
        - 救出村民数量: +20
        - 村长女儿: 死亡
      长期后果:
        - 村长好感度: -50
        - 村民好感度: +30
        - 解锁: "人民英雄"成就
      跳转: node_east_001
      
    选项B:
      文本: "去西边，救出村长的女儿"
      条件: 无
      立即后果:
        - 救出村民数量: +1
        - 村长女儿: 存活
      长期后果:
        - 村长好感度: +50
        - 村民好感度: -10
        - 解锁: 村长女儿成为同伴
      跳转: node_west_001
      
    选项C:
      文本: "尝试两全其美"
      条件: 敏捷 ≥ 15 或 有瞬移能力
      立即后果:
        - 救出村民数量: +15
        - 村长女儿: 存活
      长期后果:
        - 村长好感度: +30
        - 村民好感度: +20
        - 解锁: "完美救援"成就
      跳转: node_perfect_001
      
  默认选择:
    条件: 5秒内未选择
    结果: 随机选择A或B
```

### 后果系统

```yaml
后果类型:
  立即后果:
    时间: 选择后立即发生
    表现: 画面、对话、状态变化
    示例: NPC死亡、获得物品
    
  短期后果:
    时间: 几分钟到几小时内
    表现: 任务变化、对话变化
    示例: 任务目标改变
    
  长期后果:
    时间: 几小时到整个游戏
    表现: 剧情分支、结局变化
    示例: 阵营关系、结局类型
    
  隐藏后果:
    时间: 不可预测
    表现: 玩家不易察觉
    示例: 影响后续游戏的彩蛋
    
后果追踪:
  变量系统:
    - 道德值: -100 到 +100
    - 关系值: 每个NPC独立追踪
    - 事件标记: 关键选择的布尔值
    
  示例:
    选择: 救了村民A
    标记: saved_villager_a = true
    后续影响:
      - 村民A在第三章出现
      - 村民A提供帮助
      - 村民A的敌人成为你的敌人
```

## 叙事工具

### 叙事变量表

```yaml
叙事变量:
  全局变量:
    player_name: 玩家名称
    karma: 道德值
    reputation: 声望值
    
  角色变量:
    npc_{id}_affection: 好感度
    npc_{id}_trust: 信任度
    npc_{id}_alive: 存活状态
    
  事件变量:
    event_{id}_completed: 事件完成
    event_{id}_outcome: 事件结果
    event_{id}_method: 完成方式
    
  物品变量:
    item_{id}_owned: 拥有状态
    item_{id}_used: 使用状态
```

### 条件判断系统

```yaml
条件类型:
  简单条件:
    格式: variable operator value
    示例: karma >= 50
    
  复合条件:
    格式: condition AND/OR condition
    示例: karma >= 50 AND saved_village == true
    
  脚本条件:
    格式: 自定义函数
    示例: hasItem("magic_sword") && getLevel() >= 10
    
条件应用:
  对话解锁:
    条件: affection_ella >= 30
    结果: 解锁艾拉的隐藏对话
    
  分支解锁:
    条件: karma <= -50
    结果: 解锁邪恶路线
    
  结局判定:
    条件: saved_all_villages && defeated_demon_lord
    结果: 真结局
```

## 结局设计

```yaml
结局系统:
  结局类型:
    真结局:
      条件: 完成所有关键任务，道德值高
      描述: 完美的结局，所有问题解决
      
    普通结局:
      条件: 完成主线，未达成特殊条件
      描述: 标准结局，有遗憾但也有希望
      
    坏结局:
      条件: 道德值低或关键失败
      描述: 悲剧结局，主角失败
      
    隐藏结局:
      条件: 特殊条件组合
      描述: 意想不到的结局
      
  结局组合:
    主结局: 由主要选择决定
    角色结局: 由角色关系决定
    世界结局: 由世界状态决定
    
  结局展示:
    - 回顾关键选择
    - 展示后果影响
    - 角色命运交代
    - 世界未来展望
```
