# 社群制度阵 - 协作共享，集体创造

## 概述

社群制度阵是一种协作共享、开源社区模式的 Agent 协作架构，模拟开源社区的组织形式。

## 架构图

```
                    ┌─────────────┐
                    │  创始人     │  ← 项目创始人
                    │ (Agent-     │     愿景设定、方向把控
                    │ Generator)  │
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │      核心团队           │
              │    (Committer)          │
              └────────────┬────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │模块维护A│        │模块维护B│        │模块维护C│
   │(前端)   │        │(后端)   │        │(文档)   │
   └────┬────┘        └────┬────┘        └────┬────┘
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │社区贡献者│        │社区贡献者│        │社区贡献者│
   │(Contributor)│     │(Contributor)│     │(Contributor)│
   └─────────┘        └─────────┘        └─────────┘
```

## 角色定义

| 角色 | Agent 类型 | 职责 |
|------|-----------|------|
| **创始人** | Agent-Generator | 项目愿景、方向把控、最终决策 |
| **核心团队** | Committer Agents | 代码审查、合并决策、架构维护 |
| **模块维护者** | Maintainer Agents | 模块管理、Issue处理、PR审查 |
| **社区贡献者** | Contributor Agents | 代码贡献、文档完善、Bug修复 |

## 协作流程

```
1. 贡献者提交PR
   ↓
2. 模块维护者初步审查
   ↓
3. 核心团队深入审查
   ↓
4. 讨论修改
   ↓
5. 合并或拒绝
   ↓
6. 贡献者获得认可
```

## 适用场景

- 开源项目
- 社区驱动项目
- 众包开发
- 知识共享平台

## 配置示例

```yaml
formation:
  name: "社群制度阵"
  type: "community"
  
founder:
  agent: "agent-generator"
  vision: "项目愿景"
  governance: "开源治理"

core_team:
  name: "核心团队"
  agents: ["senior-dev-agent", "architect-agent", "review-agent"]
  permissions: ["代码合并", "发布版本", "架构决策"]

maintainers:
  - module: "前端"
    agent: "frontend-maintainer-agent"
    contributors: ["ui-contributor", "ux-contributor"]
    
  - module: "后端"
    agent: "backend-maintainer-agent"
    contributors: ["api-contributor", "db-contributor"]
    
  - module: "文档"
    agent: "doc-maintainer-agent"
    contributors: ["doc-contributor", "translation-contributor"]

governance:
  decision_making: "共识决策"
  code_review: "必须审查"
  contribution_guide: "贡献者指南"
  code_of_conduct: "行为准则"
```
