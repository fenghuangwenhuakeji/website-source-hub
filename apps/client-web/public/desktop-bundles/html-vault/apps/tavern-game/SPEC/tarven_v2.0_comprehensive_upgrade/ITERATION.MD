## 📌 迭代核心原则

```
requirement → DESIGN → task → requirement → DESIGN → task ...
      ↓            ↓          ↓
   需求定义     系统设计    任务分解
      ↓            ↓          ↓
   需求变更   ← 设计调整 ← 任务反馈
```

**每2-3周完成一轮迭代，循环往复，持续完善**
