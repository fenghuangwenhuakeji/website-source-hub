# v3.4 记忆中心升级记录

## 改造范围
- RAG 上下文
- 三层记忆
- IndexedDB 存储结构

## RAG 上下文
- 页面改为「选范围 -> 搜关键词 -> 生成上下文包」。
- 新增常用范围：写作续写、查设定、轻量检索、资料库。
- 上下文包固定显示在右侧，支持复制、存资料库、存工作记忆。
- 检索结果支持复制、存记忆、AI重排。
- AI摘要改为面向小说续写的压缩包：保留人物状态、世界规则、伏笔、章节事实、禁写点。
- 刷新索引入口会重新加载 RAG 文档分块。

## 三层记忆
- 首页改成直接任务入口：临时记一下、存成长期规则、生成记忆包、同步到RAG、智能整理、同步世界实体。
- 新增「记忆包」页：输入关键词和章节号后，合并工作记忆、长期记忆、RAG、实体、世界观，生成可直接给 AI 使用的上下文。
- 工作记忆支持模块、标签、优先级；单条可升长期、入RAG、复制、删除。
- 长期记忆支持模块、标签、重要度档位；单条可入RAG、复制、删除。
- 总览新增永久记忆计数。

## 底层修复
- IndexedDB 版本升级到 v13。
- 新增 `rag_documents` 存储表，RAG 文档不再只能降级存入 vectors。
- 新增 `DB.keys()`，修复写作模式统计和检索依赖。
- RAG 搜索会自动初始化文档索引，避免页面外调用时分块索引为空。
- `RAGSystem.buildContext()` 支持按当前数据源筛选构建上下文。
