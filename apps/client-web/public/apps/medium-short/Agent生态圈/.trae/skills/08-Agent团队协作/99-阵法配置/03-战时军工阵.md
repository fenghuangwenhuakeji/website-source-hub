# 战时军工阵 - 高效快速，紧急响应

## 概述

战时军工阵是一种扁平化、快速响应的 Agent 协作架构，模拟战时军事指挥体系，适合紧急项目和危机处理场景。

## 角色定义

| 角色 | Agent 类型 | 职责 |
|------|-----------|------|
| **指挥官** | Agent-Generator | 总指挥、决策、资源调配 |
| **突击一队** | Core Team | 核心功能开发 |
| **突击二队** | Feature Team | 次要功能开发 |
| **突击三队** | Fix Team | 紧急修复、Bug处理 |
| **侦察队** | QA Team | 测试、验证、监控 |
| **后勤队** | Support Team | 文档、部署、运维支持 |

## 协作流程

```
1. 指挥官接收紧急任务
   ↓
2. 快速评估，分配突击队
   ↓
3. 三队并行突击
   - 突击一队: 核心功能 (P0)
   - 突击二队: 次要功能 (P1)
   - 突击三队: 紧急修复 (Blocker)
   ↓
4. 侦察队实时测试验证
   ↓
5. 后勤队同步准备部署
   ↓
6. 快速迭代，每日多次交付
   ↓
7. 任务完成，总结复盘
```

## 适用场景

- MVP快速开发
- 紧急修复
- 危机处理
- 黑客马拉松
- 原型验证

## 优势

1. **响应快速**: 扁平结构，决策迅速
2. **并行高效**: 多队同时突击
3. **灵活机动**: 可随时调整优先级
4. **目标明确**: 聚焦核心目标

## 配置示例

```yaml
formation:
  name: "战时军工阵"
  type: "emergency"
  
teams:
  assault_1:
    name: "突击一队"
    priority: "P0"
    agents: ["core-dev-agent", "architect-agent"]
    tasks: ["核心功能", "架构搭建"]
    
  assault_2:
    name: "突击二队"
    priority: "P1"
    agents: ["feature-dev-agent", "ui-agent"]
    tasks: ["次要功能", "界面开发"]
    
  assault_3:
    name: "突击三队"
    priority: "Blocker"
    agents: ["debug-agent", "hotfix-agent"]
    tasks: ["Bug修复", "性能优化"]
    
  recon:
    name: "侦察队"
    agents: ["test-agent", "monitor-agent"]
    tasks: ["测试验证", "监控告警"]
    
  logistics:
    name: "后勤队"
    agents: ["doc-agent", "deploy-agent"]
    tasks: ["文档编写", "部署准备"]

workflow:
  standup: "每日3次站会(早中晚)"
  delivery: "每日多次交付"
  review: "实时代码审查"
```
