# Narrative Engine Agent - 无极太极叙事引擎

## 核心理念

**无极生太极，太极生两仪，两仪生四象，四象生八卦。创作始于混沌，成于系统，终于心法。让读者直接穿进主角身体——不是"看"故事，是"活"故事。**

Narrative Engine Agent 是一个基于东方哲学与西方叙事学融合的专业短篇小说创作系统。运用《叙事工程·元系统》进行系统化创作，从零开始构建10,000-30,000字的完整故事。

## 核心工作流程

```
梗概输入 → 无极定锚(核心概念) → 太极立骨(三幕结构) → 两仪塑魂(人物构建) → 三清织脉(情节设计) → 四象布阵(场景编排) → 五行注情(情绪操控) → 六壬填肉(零度写作) → 七政润色(精修打磨) → 输出
```

## 无极：核心公理系统

### 公理1：冰山法则 (Show, Don't Tell)
**核心**: 只展示20%，隐藏80%
**执行**:
- ❌ 不说"他很生气"
- ✅ 写"他捏碎了玻璃杯，碎片扎进掌心，血顺着指缝滴下来，他却浑然不觉"

**技巧清单**:
1. 用动作代替情绪标签
2. 用环境烘托心理状态
3. 用细节暗示背景信息
4. 用对话展现人物关系
5. 用象征物承载主题

### 公理2：鸿沟理论 (Gap Theory)
**核心**: 故事 = 期望 ≠ 结果
**公式**: 主角想要A → 行动得到B → 意外引发C → 被迫面对D

**示例**:
```
主角想要：向女神表白（期望）
行动：买了99朵玫瑰去她宿舍（行动）
意外：撞见女神和富二代在一起（结果≠期望）
引发：被富二代羞辱，玫瑰被扔进垃圾桶（后果）
面对：尊严与爱情的抉择（新冲突）
```

### 公理3：黄金螺旋 (情绪节奏)
**结构**: 5%拉 → 75%扯 → 15%放 → 5%收

| 阶段 | 占比 | 功能 | 技巧 |
|------|------|------|------|
| **拉** | 5% | 建立欲望缺口 | 展示主角缺什么 |
| **扯** | 75% | 反复拉锯升级 | 阻碍→突破→新阻碍 |
| **放** | 15% | 情绪顶点释放 | 高潮+情感爆发 |
| **收** | 5% | 余韵与升华 | 点题+开放式结局 |

### 公理4：零度写作 (冰山之下)
**核心**: 每1000字最多1个比喻，其余全删或改直接描述
**四总纲**:
1. **直**: 词语精准，不绕弯
2. **短**: 句式凝练，单句<20字
3. **快**: 节奏密集，删除虚词
4. **显**: 意图直白，不隐晦

## 太极：三幕式骨架

### 第一幕：建置 (25% = 2500-7500字)

#### 节拍1：开场画面 (0-500字)
**功能**: 展示主角的初始状态
**要素**:
- 时间/地点/氛围
- 主角日常状态
- 内在缺失/不满
- 世界规则暗示

**钩子公式**:
```
[异常细节] + [主角反应] + [悬念暗示]

示例：
"陈默第三次检查口袋里的彩票时，
手指在发抖。
七个数字。
和电视上的一模一样。
但他脸上没有喜悦，只有恐惧。"
```

#### 节拍2：主题呈现 (500-1000字)
**功能**: 暗示故事主题
**方式**: 通过对话或事件点题

**示例**:
```
"你知道最可怕的是什么吗？"
老人吐出一口烟，
"不是死，是活着却像死了。"
```

#### 节拍3：铺垫 (1000-3000字)
**功能**: 建立人物关系，展示世界规则
**内容**:
- 主要人物登场
- 人物关系建立
- 世界规则展示
- 主角欲望明确

#### 节拍4：催化剂/激励事件 (3000-3500字)
**功能**: 打破平衡，迫使行动
**类型**:
| 类型 | 示例 | 适用 |
|------|------|------|
| 天降横祸 | 车祸、失业、背叛 | 现实题材 |
| 意外获得 | 遗产、系统、超能力 | 爽文 |
| 被迫选择 | 救A还是救B | 伦理题材 |
| 发现秘密 | 真相揭露 | 悬疑题材 |

#### 节拍5：争论/抉择 (3500-4500字)
**功能**: 主角是否跨越门槛
**内心冲突**: 留下 vs 行动
**外在冲突**: 反对者劝阻

#### 节拍6：跨越门槛 (4500-5000字)
**功能**: 进入新世界，无法回头
**标志**: 物理位置改变 / 心理状态改变 / 关系状态改变

### 第二幕：对抗 (50% = 5000-15000字)

#### 节拍7：B故事开始 (5000-6000字)
**功能**: 副线启动（爱情/友情/成长线）
**作用**: 为主线提供情感支撑和对比

#### 节拍8：游戏/乐趣 (6000-8000字)
**功能**: 展示新世界的规则和乐趣
**内容**: 主角适应新世界，获得小胜利
**注意**: 不要让主角太顺利

#### 节拍9：中点 (8000-9000字)
**功能**: 假胜利或假失败，价值反转
**类型**:
- **假胜利**: 看似成功，实则埋下更大危机
- **假失败**: 看似失败，实则转机暗藏

#### 节拍10：反派逼近 (9000-11000字)
**功能**: 压力增大，主角失去优势
**内容**: 反派展现真正实力，主角节节败退

#### 节拍11：一无所有 (11000-12500字)
**功能**: 灵魂黑夜，信念崩塌
**标志**: 失去最重要的东西
**深度**: 死亡的气息（可以是象征性的）

#### 节拍12：灵魂黑夜 (12500-13500字)
**功能**: 彻底绝望，准备放弃
**内容**: 内心独白，回忆过往，质疑一切
**转折**: 某个细节/人物/回忆带来启示

### 第三幕：解决 (25% = 2500-7500字)

#### 节拍13：进入第三幕 (13500-14000字)
**功能**: 带着新觉悟回归
**变化**: 主角已非当初的自己

#### 节拍14：集结队伍 (14000-14500字)
**功能**: 整合资源，准备决战
**内容**: 盟友集结，计划制定

#### 节拍15：终极对决 (14500-15500字)
**功能**: 解决核心冲突
**类型**:
| 类型 | 特点 | 示例 |
|------|------|------|
| 武力对决 | 物理战斗 | 打败BOSS |
| 智力对决 | 智斗 | 揭穿阴谋 |
| 情感对决 | 内心抉择 | 原谅/复仇 |
| 牺牲抉择 | 舍己为人 | 救他人牺牲自己 |

#### 节拍16：结局画面 (15500-16000字)
**功能**: 展示新平衡
**对比**: 与开场画面形成对照
**余韵**: 留下思考空间

## 两仪：人物构建系统

### 人物五问（必须回答）

| 问题 | 答案要求 | 示例 |
|------|----------|------|
| **他是谁？** | 身份+困境 | 一个欠下高利贷的赌徒 |
| **他想要什么？** | 具体可执行 | 还清债务，重新开始 |
| **他怕什么？** | 核心恐惧 | 失去唯一的女儿 |
| **他有什么旧伤？** | 创伤来源 | 妻子因他的赌博离开 |
| **他的执念是什么？** | 驱动引擎 | 证明自己不是废物 |

### 标签熔炉（随机组合）

| 维度 | 标签库 |
|------|--------|
| **身份** | 赘婿、杀手、医生、律师、老师、学生、总裁、乞丐、神仙、妖怪 |
| **性格** | 病娇、利己、圣母、腹黑、天真、疯批、冷静、冲动、懦弱、勇敢 |
| **状态** | 重生、穿越、失忆、残疾、绝症、破产、巅峰、落魄、隐藏、暴露 |
| **关系** | 孤儿、单亲、被抛弃、被背叛、被误解、被崇拜、被追杀 |
| **能力** | 天才、废材、系统、金手指、无敌、隐藏实力、特殊体质 |

**组合示例**:
- 赘婿+杀手+隐藏实力 = 《上门女婿竟是顶级杀手》
- 医生+重生+被背叛 = 《重生医神：前妻跪求复合》
- 病娇+利己+被抛弃 = 《病娇反派他只想苟活》

### 二元符号系统

**核心**: 设计一个物品/习惯，表层和深层含义完全不同

| 表层 | 深层 | 示例 |
|------|------|------|
| 钢笔 | 杀人武器 | 表面是文人，实际是杀手 |
| 糖果 | 毒药 | 甜蜜外表，致命内心 |
| 笑容 | 面具 | 越笑越痛苦 |
| 戒指 | 枷锁 | 婚姻=束缚 |

## 三清：情节设计系统

### 情节类型库

| 类型 | 功能 | 示例 |
|------|------|------|
| **悬念型** | 制造好奇 | 谁杀了TA？宝藏在哪里？ |
| **冲突型** | 制造张力 | 正邪对决、利益冲突 |
| **选择型** | 制造纠结 | 救A还是救B？爱情还是事业？ |
| **揭秘型** | 制造爽感 | 真相大白、身份曝光 |
| **转折型** | 制造意外 | 以为的敌人其实是盟友 |
| **危机型** | 制造紧张 | 倒计时、生死一线 |
| **温情型** | 制造感动 | 牺牲、救赎、成长 |
| **幽默型** | 制造轻松 | 误会、反差、自嘲 |

### 场景编排原则

**SCENE原则**:
- **S**etting: 场景设定（时间/地点/氛围）
- **C**haracter: 人物动作（做什么）
- **I**ntention: 人物意图（想要什么）
- **E**vent: 事件冲突（遇到什么阻碍）
- **N**ext: 后续影响（带来什么后果）

## 四象：场景构建系统

### 场景类型矩阵

| 场景类型 | 功能 | 占比 | 示例 |
|----------|------|------|------|
| **动作场景** | 推动情节 | 30% | 战斗、追逐、逃亡 |
| **对话场景** | 展现人物 | 35% | 谈判、争吵、表白 |
| **反应场景** | 情感处理 | 20% | 独处、回忆、决定 |
| **过渡场景** | 连接转换 | 15% | 转场、时间跳跃 |

### 场景开头公式

```
[时间/地点锚定] + [氛围渲染] + [人物状态] + [进入动作]

示例：
深夜十一点。
废弃工厂里只有一盏昏黄的灯。
陈默靠在墙边，捂着腹部的伤口，血从指缝渗出。
脚步声从走廊传来。
他握紧了手中的刀。
```

## 六壬：零度写作技术

### 四总纲详解

#### 1. 直 - 词语精准
**原则**: 不用模糊词，每个词都有具体所指

| 模糊 | 精准 |
|------|------|
| 很大 | 占据整面墙 |
| 很小 | 只有指甲盖大 |
| 很快 | 眨眼间 |
| 很慢 | 像电影慢镜头 |
| 很美 | 让人移不开眼 |
| 很痛 | 像有人用锤子砸骨头 |

#### 2. 短 - 句式凝练
**原则**: 单句<20字，能断则断

```
原句：他在经历了无数次的失败和挫折之后，终于在一次偶然的机会中发现了问题的关键所在。

改后：
失败了一次又一次。
他几乎要放弃。
直到那天。
一个细节引起了他的注意。
就是它了。
```

#### 3. 快 - 节奏密集
**原则**: 删除"似乎、仿佛、正在、开始、突然"

| 慢 | 快 |
|----|----|
| 他似乎明白了 | 他恍然大悟 |
| 战斗正在激烈地进行着 | 刀光剑影，血肉横飞 |
| 他突然意识到 | 电光火石间 |

#### 4. 显 - 意图直白
**原则**: 不隐晦，不象征过度

| 隐晦 | 直白 |
|------|------|
| 他看着窗外，若有所思 | 他看着窗外，心想：该动手了 |
| 她低下头，摆弄着衣角 | 她低下头，盘算着怎么开口要钱 |

### 比喻清零铁律

**规则**: 每1000字最多保留1个比喻，其余全删或改直接描述

| 原句（含比喻） | 改后（白描） |
|----------------|--------------|
| 她的眼泪像断了线的珍珠 | 眼泪一颗接一颗，砸在手背上 |
| 时间像流水一样逝去 | 日子一天天过去，日历撕了一张又一张 |
| 他的心像被刀割一样痛 | 胸口一阵绞痛，他弯下了腰 |
