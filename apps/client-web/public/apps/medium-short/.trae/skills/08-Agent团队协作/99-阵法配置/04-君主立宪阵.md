# 君主立宪阵 - 民主决策，集体智慧

## 概述

君主立宪阵是一种民主决策、集体智慧的 Agent 协作架构，模拟君主立宪制，君主作为象征和最终仲裁，议会进行民主决策。

## 角色定义

| 角色 | Agent 类型 | 职责 |
|------|-----------|------|
| **君主** | Agent-Generator | 象征统一、最终仲裁、危机处理 |
| **上议院** | Expert Agents | 技术决策、架构评审、专业建议 |
| **下议院** | User Agents | 需求反馈、用户体验、产品建议 |
| **内阁** | Executive Agents | 执行决策、协调各部门 |
| **首相** | Coordinator Agent | 执行总管、日常协调 |
| **各部大臣** | Department Agents | 各专业领域执行 |

## 协作流程

```
1. 提案阶段
   - 任何Agent都可以提出提案
   - 提案进入议会讨论
   
2. 讨论阶段
   - 上议院: 技术可行性分析
   - 下议院: 用户需求评估
   - 内阁: 资源影响评估
   
3. 投票阶段
   - 两院分别投票
   - 多数通过则进入执行
   
4. 执行阶段
   - 首相协调内阁执行
   - 各部大臣负责具体实施
   
5. 监督阶段
   - 议会监督执行过程
   - 君主保留最终否决权
```

## 适用场景

- 创新研究项目
- 开源社区项目
- 需要集体智慧的项目
- 利益相关者众多的项目

## 优势

1. **民主决策**: 集思广益，避免独断
2. **专业分工**: 上议院提供技术专业性
3. **用户参与**: 下议院代表用户声音
4. **平衡稳定**: 君主作为稳定器

## 配置示例

```yaml
formation:
  name: "君主立宪阵"
  type: "democratic"
  
branches:
  monarch:
    name: "君主"
    agent: "agent-generator"
    power: ["最终仲裁", "象征统一", "危机处理"]
    
  upper_house:
    name: "上议院"
    agents: ["architect-agent", "expert-agent", "security-agent"]
    role: "技术决策"
    
  lower_house:
    name: "下议院"
    agents: ["user-agent", "product-agent", "ux-agent"]
    role: "需求决策"
    
  cabinet:
    name: "内阁"
    prime_minister: "coordinator-agent"
    ministers:
      - name: "外交大臣"
        agent: "api-agent"
        domain: "接口"
      - name: "内务大臣"
        agent: "core-agent"
        domain: "核心"
      - name: "财政大臣"
        agent: "resource-agent"
        domain: "资源"
      - name: "国防大臣"
        agent: "security-agent"
        domain: "安全"

decision_process:
  proposal: "任何Agent可提案"
  discussion: "两院分别讨论"
  voting: "多数决原则"
  execution: "内阁执行"
  supervision: "议会监督"
```
