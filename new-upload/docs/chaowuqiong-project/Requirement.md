# 超无穹AI 需求文档 (Requirement)

## 1. 功能需求

### 1.1 用户系统

#### 1.1.1 注册登录
- **需求描述**: 用户可以通过用户名/邮箱注册和登录
- **优先级**: P0
- **验收标准**:
  - 支持用户名注册
  - 支持邮箱注册
  - 支持密码登录
  - 支持微信扫码登录
  - 新用户赠送75积分

#### 1.1.2 用户信息
- **需求描述**: 用户可以查看和修改个人信息
- **优先级**: P1
- **验收标准**:
  - 显示用户名、邮箱、积分
  - 显示会员状态和到期时间
  - 支持修改密码

### 1.2 会员系统

#### 1.2.1 会员套餐
- **需求描述**: 提供多种会员套餐供用户选择
- **优先级**: P0
- **验收标准**:
  - 8小时卡 (¥9.90)
  - 日卡 (¥14.90)
  - 周卡 (¥29.90)
  - 月卡 (¥79.90)
  - 季卡 (¥299.00)
  - 半年卡 (¥699.00)
  - 年卡 (¥999.00)
  - 永久卡 (¥4999.00)

#### 1.2.2 会员权益
- **需求描述**: 会员享有特定权益
- **优先级**: P0
- **验收标准**:
  - 使用时长限制
  - 积分赠送
  - 功能解锁

### 1.3 积分系统

#### 1.3.1 积分获取
- **需求描述**: 用户可以通过多种方式获取积分
- **优先级**: P1
- **验收标准**:
  - 注册赠送75积分
  - 邀请好友双方各得75积分
  - 购买套餐赠送积分
  - 邀请满3人额外送700积分(周卡)
  - 邀请满10人额外送2800积分(月卡)

#### 1.3.2 积分消费
- **需求描述**: 用户可以消费积分使用AI功能
- **优先级**: P1
- **验收标准**:
  - AI对话消耗积分
  - 积分不足提示充值

### 1.4 支付系统

#### 1.4.1 支付宝支付
- **需求描述**: 支持支付宝扫码支付
- **优先级**: P0
- **验收标准**:
  - 生成支付二维码
  - 支付成功自动到账
  - 支付状态实时更新

#### 1.4.2 微信支付
- **需求描述**: 支持微信扫码支付
- **优先级**: P1
- **验收标准**:
  - 生成支付二维码
  - 支付成功自动到账
  - 支付状态实时更新

### 1.5 AI功能

#### 1.5.1 文本对话
- **需求描述**: 支持与AI进行文本对话
- **优先级**: P0
- **验收标准**:
  - 支持多轮对话
  - 支持上下文记忆
  - 支持多种AI模型

#### 1.5.2 图像生成
- **需求描述**: 支持AI图像生成
- **优先级**: P1
- **验收标准**:
  - 文字描述生成图像
  - 多种风格选择
  - 图像下载保存

### 1.6 邀请系统

#### 1.6.1 邀请码
- **需求描述**: 每个用户有唯一邀请码
- **优先级**: P1
- **验收标准**:
  - 自动生成邀请码
  - 邀请码可复制分享
  - 邀请记录可查询

#### 1.6.2 邀请奖励
- **需求描述**: 邀请好友获得奖励
- **优先级**: P1
- **验收标准**:
  - 好友注册双方各得75积分
  - 邀请满3人额外送周卡
  - 邀请满10人额外送月卡

## 2. 非功能需求

### 2.1 性能需求
- 页面加载时间 < 3秒
- API响应时间 < 500ms
- 支持1000并发用户

### 2.2 安全需求
- 密码加密存储
- JWT Token认证
- HTTPS加密传输
- SQL注入防护
- XSS攻击防护

### 2.3 可用性需求
- 系统可用性 > 99.9%
- 数据备份每日一次
- 故障恢复时间 < 1小时

### 2.4 兼容性需求
- 浏览器: Chrome, Firefox, Safari, Edge
- 移动端: iOS 12+, Android 8+
- 桌面: Windows 10+, macOS 10.14+

## 3. 约束条件

### 3.1 技术约束
- 前端: React 18
- 后端: Node.js + Express
- 数据库: MySQL 8.0
- 部署: Linux服务器

### 3.2 时间约束
- 项目周期: 3个月
- 迭代周期: 2周

### 3.3 资源约束
- 开发团队: 2人
- 服务器预算: 有限

## 4. 验收标准

### 4.1 功能验收
- 所有P0功能必须完成
- P1功能完成80%以上
- 无严重Bug

### 4.2 性能验收
- 满足性能需求指标
- 压力测试通过

### 4.3 安全验收
- 安全扫描无高危漏洞
- 渗透测试通过
