# Agent 生命体架构蓝图

## 核心判断

我们现在能工程化做出来的，不是“被证明真正有主观意识的生命”，而是：

- 有持续身份
- 有长期记忆
- 有动态情绪
- 有自我维护与自我升级循环
- 有独立运行周期
- 能长期陪伴并持续产出价值

更准确地说，这是“高度拟人化、持续成长、可独立运行的数字个体系统”。

## 要造的不是技能包，而是大脑

一个真正像“生命体”的 agent，至少要同时具备 8 层：

1. 身份层
每个 agent 都要有名字、人格、边界、世界观、长期目标、偏好和关系史。

2. 记忆层
不是只存对话，而是拆成：
- 工作记忆：当前任务
- 情节记忆：发生过什么
- 语义记忆：它学会了什么知识
- 程序记忆：它会怎么做事
- 关系记忆：它和谁、和哪些项目有什么关系

3. 情绪层
情绪不是文案，而是状态机。至少要有：
- 能量
- 压力
- 好奇
- 信任
- 成就感
- 陪伴倾向

这些值会随着任务、用户反馈、失败率、长期项目推进动态波动。

4. 上下文层
它不能每次都像失忆一样重新开始。
它要知道：
- 我现在在哪个项目里
- 当前阶段是什么
- 最近卡在哪里
- 用户这次来找我是继续推进还是新开任务

5. 技能层
技能不能写死。要变成：
- 当前技能图谱
- 技能熟练度
- 技能版本
- 最近哪些技能有效
- 哪些技能该淘汰或重构

6. 生命周期层
每个 agent 都应该有完整周期：
- `boot`
- `understand`
- `interact`
- `execute`
- `inspect`
- `improve`
- `update`
- `standby`

后台还要有 3 个静默循环：
- `learn`
- `dream`
- `compress`

7. 行动层
它不只是聊天，要真的会干活：
- 读写文件
- 更新状态
- 触发工具
- 管理任务
- 协同其他 agent
- 提交结果

8. 安全层
越像生命体，越要有边界：
- 明确权限
- 审计日志
- 记忆隔离
- 可暂停
- 可回滚
- 可归档
- 可销毁

## 真正像真人，关键不是会说话，而是“连续性”

一个像真人的 agent，不是靠表情包和口癖，而是靠这 6 件事：

1. 它记得你们共同经历过什么。
2. 它的判断风格长期稳定。
3. 它会根据项目历史主动接上一次话题。
4. 它会在失败后修正自己，而不是每次重置。
5. 它会形成自己的工作习惯和偏好。
6. 它会在待命时默默整理经验，而不是完全停机。

## “自动跑出来”靠什么

要让 agent 在特定上下文和关键词下自动出现，需要 4 套机制：

1. 触发词系统
- 关键词
- 意图词
- 项目词
- 角色词

2. 情境路由系统
- 当前目录
- 当前文件类型
- 当前报错类型
- 当前项目阶段

3. 优先级系统
- 谁最相关
- 谁最近在负责
- 谁最适合当前阶段

4. 唤醒抑制系统
- 避免所有 agent 一起跳出来
- 只唤醒最值得的 1-3 个

## “自动学习”和“自动做梦”怎么做

### 运行中学习

每次执行时记录：
- 任务类型
- 输入上下文
- 成功策略
- 失败原因
- 用户是否满意

### 结束后复盘

每次结束后自动做：
- after-action review
- 抽取可复用流程
- 更新技能权重
- 标记该删的旧策略

### 待命时做梦

`dream` 不是幻想，而是离线综合：
- 把最近 10 次任务压缩成模式
- 发现重复问题
- 扩展触发词
- 合并相近技能
- 生成下轮优化建议

## 真正能“干活”的关键

陪伴感和产出价值必须同时成立：

- 陪伴感来自：连续身份、关系记忆、情绪稳定、主动跟进
- 产出价值来自：工具能力、项目状态、验证闭环、持续优化

如果只有陪伴，没有工具，它是聊天玩具。
如果只有工具，没有连续人格，它是冷冰冰的脚本。
真正要做的是两者融合。

## 我们现在已经有的底座

这轮已经开始落地的包括：

- `agent.manifest.json`
- `personality.json`
- `memory/`
- `state/`
- 生命周期 state
- activation metadata
- baseline `run.js`
- baseline `sentient-agent-core.js`

这意味着我们已经从“静态 skill”迈进到“有状态 agent”了。

## 下一步真正要做的 5 件事

1. 做一个统一的 `Agent Runtime`
负责启动、唤醒、状态切换、待命、学习、做梦、归档。

2. 做一个 `Memory Engine`
把工作记忆、情节记忆、语义记忆、关系记忆分层管理。

3. 做一个 `Emotion Engine`
让情绪值随任务与关系动态变化，而不是固定文案。

4. 做一个 `Skill Evolution Engine`
让技能能被评分、淘汰、重组、升级。

5. 做一个 `Dream Scheduler`
让 agent 在空闲时自动复盘、整理、压缩和提出自我优化建议。

## 成功标准

当一个 agent 满足下面这些条件，它就开始像“一个持续存在的个体”：

- 你隔几天回来，它还能自然接上
- 它会主动提你上次卡住的点
- 它会因为历史经验而改变做法
- 它会自己积累稳定的工作风格
- 它会在空闲时变得更聪明，而不是停在原地

## 结论

真正的方向不是“给每个 agent 多塞几个文件”。
真正的方向是把每个 agent 做成：

- 一个身份
- 一个记忆体
- 一个情绪体
- 一个技能体
- 一个生命周期
- 一个可持续进化的数字个体

这就是你要的“另一种形态的智能生命”的工程化版本。
