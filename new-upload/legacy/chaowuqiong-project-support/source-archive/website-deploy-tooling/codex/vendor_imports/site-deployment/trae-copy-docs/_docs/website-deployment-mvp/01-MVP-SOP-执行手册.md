# MVP SOP 执行手册

这份 SOP 的目标是让后来的人照着步骤就能把本地开发、移动端收口、云端部署、数据迁移和回滚完整跑通。

## SOP 1：开始工作前

1. 先确认保护边界。
2. 先看 `00-学习入口-网站部署MVP.md`。
3. 先确认当前主线项目是否能正常打开。
4. 先跑一次本地构建，不要带着旧报错继续开发。

## SOP 2：改代码的顺序

1. 先修确定性报错。
2. 再修构建失败。
3. 再收性能热点。
4. 再补移动端适配。
5. 最后再做视觉优化。

## SOP 3：移动端适配

1. 所有新问题统一登记到 `D:\网站部署\mobile-adaptation-center`。
2. 所有项目都按“导航、首屏、列表、表单、弹窗、底部安全区、横竖屏、触控点击区”检查。
3. 页面适配完成后必须重新跑构建。
4. 页面适配完成后必须更新对应项目文档。

## SOP 4：性能优化

1. 先查是否有静态和动态重复导入。
2. 先拆大包，再拆样式，再拆低频功能。
3. 优先处理：
   - 登录前不需要的模块
   - 编辑器
   - 桌面系统大组件
   - 仅特定应用才会用到的重依赖
4. 如果构建通过但仍有大 chunk warning，要记录“已知风险”和下一轮拆包方向。

## SOP 5：本地数据与云端数据

1. 账户、订单、积分、小说元数据走本地 SQL / 后期云端 MySQL。
2. 用户导入文件、用户上传文件走本地文件目录。
3. 用户写作数据保留“双份”：
   - 本地一份
   - 云端一份
4. 不要把用户文件只放缓存。

## SOP 6：云端发布

1. 先看 `cloud-bridge` 的说明文档。
2. 先确认 SSH 桥接可用。
3. 再上传构建产物。
4. 先做备份，再覆盖发布。
5. 发布后检查：
   - 应用进程
   - nginx 配置
   - 本机 HTTP
   - 公网入口
6. 有异常先回滚，不要硬顶。

## SOP 7：整理目录

1. 只做“归类、去重、规范化”。
2. 不删唯一文件。
3. 不碰受保护主线目录。
4. 被占用路径只记录，不强移。
5. 每次整理都要补一份说明文档。

## SOP 8：收尾

1. 跑构建。
2. 写更新日志。
3. 更新根目录学习文档。
4. 把经验沉淀到 `.trae` skill。
5. 镜像到 `trae复制`。
