# 人民民主阵 - 平等参与，广泛征集

## 概述

人民民主阵是一种平等参与、民主投票的 Agent 协作架构，模拟人民代表大会制度。

## 架构图

```
                    ┌─────────────────────┐
                    │   人民代表大会     │  ← 最高决策机构
                    │ (Decision Council) │
                    └──────────┬──────────┘
                               │
            ┌──────────────────┼──────────────────┐
            │                  │                  │
       ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
       │技术代表团│        │产品代表团│        │用户代表团│
       └────┬────┘        └────┬────┘        └────┬────┘
            │                  │                  │
       ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
       │开发代表 │        │设计代表 │        │终端用户 │
       │架构代表 │        │产品代表 │        │企业用户 │
       │测试代表 │        │运营代表 │        │开发者   │
       └─────────┘        └─────────┘        └─────────┘
```

## 角色定义

| 角色 | Agent 类型 | 职责 |
|------|-----------|------|
| **人民代表大会** | Decision Council | 最高决策、重大事项投票 |
| **技术代表团** | Tech Delegation | 技术决策、架构评审 |
| **产品代表团** | Product Delegation | 产品决策、需求评审 |
| **用户代表团** | User Delegation | 用户反馈、体验评估 |
| **各代表** | Representative Agents | 具体领域代表 |

## 协作流程

```
1. 提案提交
   ↓
2. 各代表团讨论
   ↓
3. 人民代表大会投票
   ↓
4. 多数通过则执行
   ↓
5. 执行监督
```

## 适用场景

- 众包项目
- 社区决策
- 多方利益相关者项目
- 需要广泛共识的项目

## 配置示例

```yaml
formation:
  name: "人民民主阵"
  type: "democratic"
  
council:
  name: "人民代表大会"
  agents: ["decision-agent", "voting-agent"]
  authority: ["重大事项决策", "预算审批", "方向把控"]

delegations:
  tech:
    name: "技术代表团"
    representatives:
      - "开发代表"
      - "架构代表"
      - "测试代表"
    weight: 0.4
    
  product:
    name: "产品代表团"
    representatives:
      - "设计代表"
      - "产品代表"
      - "运营代表"
    weight: 0.3
    
  user:
    name: "用户代表团"
    representatives:
      - "终端用户"
      - "企业用户"
      - "开发者"
    weight: 0.3

voting:
  threshold: 0.6  # 60%通过
  method: "加权投票"
  transparency: "公开透明"
```
