# 三省六部阵 - 分工明确，流程规范

## 概述

三省六部阵是一种职能分工明确的 Agent 协作架构，模拟古代中国的三省六部制度，形成决策、审核、执行的标准化流程。

## 角色定义

| 部门 | Agent 类型 | 职责 |
|------|-----------|------|
| **中书省** | Plan Agent | 决策、规划、制定方案 |
| **门下省** | Spec Agent | 审核、把关、质量监督 |
| **尚书省** | 执行总管 | 执行、协调、进度管理 |
| **吏部** | HR Agent | 人员管理、权限分配 |
| **户部** | Resource Agent | 资源管理、成本控制 |
| **礼部** | Doc Agent | 文档管理、知识库 |
| **兵部** | Ops Agent | 运维、部署、监控 |
| **刑部** | Compliance Agent | 合规检查、代码审查 |
| **工部** | Dev Agent | 开发实施、代码生成 |

## 协作流程

```
1. 中书省(Plan) 制定方案和计划
   ↓
2. 提交门下省(Spec) 审核
   ↓
3. 审核通过 → 尚书省分配任务
   审核不通过 → 退回中书省修改
   ↓
4. 尚书省协调六部执行
   - 吏部: 分配人员权限
   - 户部: 分配资源预算
   - 礼部: 准备文档模板
   - 兵部: 准备运维环境
   - 刑部: 制定合规标准
   - 工部: 执行开发任务
   ↓
5. 六部并行工作，定期汇报尚书省
   ↓
6. 完成后提交门下省验收
   ↓
7. 验收通过 → 交付皇帝
```

## 适用场景

- 政府项目
- 大型企业系统
- 需要严格流程控制的场景
- 合规要求高的项目

## 优势

1. **流程规范**: 决策-审核-执行的标准流程
2. **分工明确**: 六部各司其职，专业高效
3. **质量保障**: 审核机制确保输出质量
4. **可追溯性**: 每个环节都有记录和责任人

## 配置示例

```yaml

  name: "三省六部阵"
  type: "functional"
  
workflow:
  decision: "中书省"
  review: "门下省"
  execution: "尚书省"
  
departments:
  - name: "吏部"
    function: "personnel"
    agents: ["permission-agent", "role-agent"]
    
  - name: "户部"
    function: "resource"
    agents: ["budget-agent", "resource-agent"]
    
  - name: "礼部"
    function: "documentation"
    agents: ["doc-agent", "knowledge-agent"]
    
  - name: "兵部"
    function: "operations"
    agents: ["deploy-agent", "monitor-agent"]
    
  - name: "刑部"
    function: "compliance"
    agents: ["audit-agent", "security-agent"]
    
  - name: "工部"
    function: "development"
    agents: ["code-agent", "test-agent"]
```
