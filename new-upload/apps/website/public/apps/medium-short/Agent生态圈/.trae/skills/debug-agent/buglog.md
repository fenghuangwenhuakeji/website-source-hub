## 按语言分类的常见缺陷模式

### JavaScript/TypeScript 常见缺陷

| 缺陷类型 | 描述 | 典型错误 | 解决方案 |
|----------|------|----------|----------|
| 类型错误 | 类型不匹配 | `TypeError: xxx is not a function` | 添加类型检查 |
| 空值访问 | 访问 undefined/null 的属性 | `Cannot read property 'x' of undefined` | 使用可选链 `?.` |
| 异步错误 | Promise 未捕获 | `UnhandledPromiseRejectionWarning` | 添加 `.catch()` |
| 闭包问题 | 循环中的闭包 | 变量引用错误 | 使用 `let` 或闭包工厂 |
| 内存泄漏 | 未清理的事件监听器 | 内存持续增长 | 移除监听器 |

### Python 常见缺陷

| 缺陷类型 | 描述 | 典型错误 | 解决方案 |
|----------|------|----------|----------|
| 导入错误 | 模块未找到 | `ModuleNotFoundError` | 安装模块或检查路径 |
| 类型错误 | 类型操作不匹配 | `TypeError` | 类型检查和转换 |
| 缩进错误 | 缩进不一致 | `IndentationError` | 统一缩进风格 |
| 编码错误 | 字符编码问题 | `UnicodeDecodeError` | 指定编码格式 |
| 依赖冲突 | 包版本冲突 | 运行时异常 | 使用虚拟环境 |

### C++ 常见缺陷

| 缺陷类型 | 描述 | 典型错误 | 解决方案 |
|----------|------|----------|----------|
| 内存泄漏 | 未释放内存 | 内存持续增长 | 使用智能指针 |
| 越界访问 | 数组越界 | `segmentation fault` | 边界检查 |
| 空指针 | 解引用空指针 | 程序崩溃 | 空指针检查 |
| 链接错误 | 符号未定义 | `undefined reference` | 检查库链接 |
| 类型不匹配 | 类型转换错误 | 编译错误 | 显式类型转换 |

### Vue3 常见缺陷

| 缺陷类型 | 描述 | 典型错误 | 解决方案 |
|----------|------|----------|----------|
| 响应式丢失 | ref/reactive 使用不当 | 数据更新视图不更新 | 正确使用 `.value` |
| 生命周期 | 钩子使用错误 | 组件行为异常 | 检查钩子顺序 |
| Props 验证 | Props 类型错误 | 警告信息 | 添加类型验证 |
| 状态管理 | 状态更新问题 | 数据不一致 | 使用正确的 mutation |
| 路由问题 | 路由配置错误 | 页面无法访问 | 检查路由配置 |

## 缺陷报告指南

### 报告前检查

- [ ] 确认是真实缺陷而非使用问题
- [ ] 搜索是否已有相同缺陷报告
- [ ] 准备完整的复现步骤
- [ ] 收集相关日志和截图
- [ ] 确认环境信息完整

### 报告质量标准

| 要素 | 要求 |
|------|------|
| 标题 | 简洁明了，包含关键信息 |
| 复现步骤 | 详细、可重复 |
| 环境信息 | 完整、准确 |
| 错误信息 | 完整日志/堆栈 |
| 预期结果 | 明确描述 |

## 缺陷处理流程

### 1. 新建缺陷

```
发现缺陷
    │
    ├── 收集信息
    │   ├── 错误日志
    │   ├── 复现步骤
    │   └── 环境信息
    │
    ├── 创建缺陷记录
    │   ├── 填写基本信息
    │   ├── 评估严重程度
    │   └── 指派处理人
    │
    └── 通知相关人员
```

### 2. 处理缺陷

```
接收缺陷
    │
    ├── 确认复现
    │   └── 验证缺陷存在
    │
    ├── 根因分析
    │   ├── 定位问题代码
    │   ├── 分析问题原因
    │   └── 确定修复方案
    │
    ├── 实施修复
    │   ├── 编写修复代码
    │   ├── 编写测试用例
    │   └── 代码审查
    │
    └── 验证修复
        ├── 单元测试
        ├── 集成测试
        └── 手动验证
```

### 3. 关闭缺陷

```
修复验证通过
    │
    ├── 更新缺陷状态
    │   └── 标记为 Resolved
    │
    ├── 记录修复信息
    │   ├── 修复方案
    │   ├── 相关 Commit
    │   └── 测试结果
    │
    └── 关闭缺陷
        └── 标记为 Closed
```
