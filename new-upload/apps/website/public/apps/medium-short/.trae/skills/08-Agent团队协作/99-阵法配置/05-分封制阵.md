# 分封制阵 - 分布式处理，区域自治

## 概述

分封制阵是一种分布式、区域自治的 Agent 协作架构，模拟古代分封制，各诸侯国（服务）拥有高度自治权，天子（网关）负责统一协调。

## 架构图

```
                    ┌─────────────┐
                    │    天子     │  ← API Gateway
                    │  (网关层)   │     统一入口、路由分发
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │  齐侯   │        │  楚侯   │        │  燕侯   │  ← 诸侯国
   │(用户服务)│        │(订单服务)│        │(商品服务)│     自治服务
   └────┬────┘        └────┬────┘        └────┬────┘
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │卿大夫   │        │卿大夫   │        │卿大夫   │
   │(子服务) │        │(子服务) │        │(子服务) │
   └────┬────┘        └────┬────┘        └────┬────┘
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │  士     │        │  士     │        │  士     │
   │(函数)   │        │(函数)   │        │(函数)   │

                           │
                    ┌──────┴──────┐
                    │   宗法制    │  ← 服务治理
                    │ (服务注册)  │     服务发现、配置中心
                    └─────────────┘
```

## 角色定义

| 角色 | Agent 类型 | 职责 |
|------|-----------|------|
| **天子** | Gateway Agent | API网关、统一入口、路由分发 |
| **齐侯** | User Service | 用户服务自治 |
| **楚侯** | Order Service | 订单服务自治 |
| **燕侯** | Product Service | 商品服务自治 |
| **赵侯** | Payment Service | 支付服务自治 |
| **魏侯** | Notification Service | 通知服务自治 |
| **卿大夫** | Sub-service Agents | 子服务、模块 |
| **士** | Function Agents | 具体函数、方法 |
| **宗法制** | Service Registry | 服务注册、发现、治理 |

## 协作流程

```
1. 请求到达天子(Gateway)
   ↓
2. 天子进行身份认证、限流检查
   ↓
3. 根据路由规则分发给对应诸侯
   ↓
4. 诸侯自治处理请求
   - 内部卿大夫协作
   - 士执行具体逻辑
   ↓
5. 如需跨诸侯协作，通过天子协调
   ↓
6. 返回结果给天子
   ↓
7. 天子统一返回给客户端
```

## 适用场景

- 微服务架构
- 分布式系统
- 多租户系统
- 需要服务自治的场景

## 优势

1. **服务自治**: 各服务独立开发、部署、扩展
2. **松耦合**: 服务间通过网关通信
3. **可扩展**: 容易添加新的诸侯(服务)
4. **容错性**: 单服务故障不影响全局

## 配置示例

```yaml
formation:
  name: "分封制阵"
  type: "distributed"
  
gateway:
  name: "天子"
  agent: "gateway-agent"
  functions: ["路由", "认证", "限流", "负载均衡"]
  
marquisates:
  - name: "齐侯"
    service: "user-service"
    agents: ["user-agent", "auth-agent", "profile-agent"]
    domain: "用户域"
    
  - name: "楚侯"
    service: "order-service"
    agents: ["order-agent", "cart-agent", "inventory-agent"]
    domain: "订单域"
    
  - name: "燕侯"
    service: "product-service"
    agents: ["product-agent", "category-agent", "search-agent"]
    domain: "商品域"
    
  - name: "赵侯"
    service: "payment-service"
    agents: ["payment-agent", "refund-agent", "billing-agent"]
    domain: "支付域"
    
  - name: "魏侯"
    service: "notification-service"
    agents: ["email-agent", "sms-agent", "push-agent"]
    domain: "通知域"

governance:
  registry: "service-registry-agent"
  config: "config-center-agent"
  monitor: "service-monitor-agent"
  
communication:
  inter_service: "通过天子协调"
  intra_service: "内部自治"
  protocol: "REST/gRPC"
```
